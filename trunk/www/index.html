Project Mural - Data Services 

Goal: To build Data Services platform as part of Project Mural as Data Services is an application of SOA pattern to the larger Data Management problem. We believe is a way of publishing master data and provides views of different instances of master data (product, customer etc.), publishing master data as a catalog service, search service etc.

 

Building Blocks

Data Services Platforms enable creation, deployment and management of services that perform the following functions

  • Universal Data Access (Relational sources, web sources, Desktop sources...)
  • Data Transformation(T of ETL)
  • Data Movement (ETL/ELT, Migration and replication)
  • Data Audit
  • Data Quality (the key)
  • Matching, Standardization, Cleansing , Profiling ...
  • Metadata Management (compliance)

Data Mashup

Demystifying Mashups 

  • Mashup a.k.a web application hybrid is a web application that combines/reuses data from more than one source (web source) into a single integrated and interactive experience.
  • Fundamental tenets of mashups is to allow the end-user to participate in building Situational Applications which are relevant to the context. What this means is  a business user could assemble a mashup by consuming services within and outside the enterprise.
  • Mashups are realization of “The Architecture of Participation” as per Web 2.0 principle.

Data Mashup Engine Architecture

 

Data Mashup Features 

  • Relational Mapping of non-relational(flatfiles, xml, webrowset) and web sources(HTML, RSS) through a mapping wizard which helps the user to map RSS Feeds, HTML tables, spreadsheets etc. as a relational table.
  • Performs Federated query of mapped relational tables through Axion Federated database
  • Provides a Netbeans 6.1 compliant Design-time Modeling environment and a Project System to manage deployment of data mashups as services
  • Run-time component is a JBI compliant service engine
  • JSON, XML, WebRowSet
  • Provides both SOAP as well as REST by leveraging  the platform (HTTPBC)
  • Can syndicate Data mashup as a Feed(part of todays demo)
  • Seamlessly integrates with platform both run-time(through JBI), design-time (Netbeans plug-in) and CASA(as can be seen in the demo)
  • Leverages HTTPBC for SOAP/REST  invocations

Mashup Database Engine

  • Embedded lightweight Federated database engine
  • Optimized for flat-file access
  • Pluggable Table types(External table interface)
  • Pluggable Index types
  • Suitable for Enterprise Data Mashups as supports adding new relational mappings with considerable ease.
  • Features a JDBC driver
